============================================================
  小红书链接注入器——使用指南 v2.6.1
============================================================

📌 一、安装插件

  在 Roche 中，进入 设置 → 插件管理 → 添加插件，粘贴以下链接：

  https://raw.githubusercontent.com/luyi90720-sys/roche-xhs-reader/main/manifest.json

  点击安装即可。安装后，在插件列表中找到「小红书链接注入器」打开即可使用。


📌 二、基本使用流程

  1. 在 Roche 中打开插件面板（点击「小红书注入器」App）
  2. 在「主面板」标签中，从下方「监听会话」列表勾选要监听的聊天（最多 3 个）
  3. 勾选「启用自动监听」开关，状态显示为「运行中」
  4. 此后，只要你在被监听会话中发送含小红书链接的消息，
     插件会自动抓取笔记内容、下载图片，替换成 AI 可读格式注入聊天

  *** 关闭插件面板后监听依然在后台运行，无需一直开着面板 ***

  *** v2.6.1 起，浏览器端默认开启内置代理，开箱即用，无需任何配置 ***


📌 三、两种运行模式

  在主面板可切换两种模式：

  模式一（直注模式）：
    - 抓取笔记全文 + 评论
    - 注入文本消息 + 独立图片消息（最多 9 张）
    - 最省事，AI 能看到完整内容和图片，但消耗 token 较多

  模式二（副 API 总结模式）：
    - 下载笔记图片 → 发给副 API（vision 多模态模型）生成 300-800 字总结
    - 注入：总结文本 + 评论 + 卡片数据
    - 不占图片 token，适合长笔记
    - 需要先在「副 API」标签页配置一个 API 预设


📌 四、配置副 API（模式二必需，模式一不需要）

  切换到「副 API」标签页：

  1. 点击「新建」
  2. 填写：
     - 预设名称：方便自己记的名字，如"默认"
     - Base URL：API 接口地址，例如 https://api.openai.com
     - API Key：你的 API 密钥
     - 模型：点击「刷新」获取可用模型列表，选择一个支持 vision 的模型，
       如 gpt-4o-mini、gpt-4o、claude-3-haiku 等
     - Temperature：保持默认 0.5 即可
  3. 点「测试连接」确认能通
  4. 点「保存预设」
  5. 回到主面板，切换到「模式二」


📌 五、代理设置（v2.6.1）

  v2.6.1 起，插件内置了官方 CF Worker 代理（默认开启），
  浏览器端开箱即用，无需任何配置。

  --- 内置代理说明 ---
  - 默认开启，浏览器端自动使用，解决 corsproxy 403 问题
  - 代理地址：https://xhs-proxy.luyi90720.workers.dev
  - 该代理仅处理小红书公开数据：笔记链接、HTML页面、小红书图片
  - 不经过任何用户隐私数据：聊天内容、角色人设、API Key 等均不经此代理
  - APK 端不受此开关影响（APK 默认走公共代理，corsproxy 在 APK 中可用）
  - 如果不放心，可以关闭内置代理，或自己部署 CF Worker 填入

  --- 关闭/开启内置代理 ---
  1. 进入「设置」标签页
  2. 找到「内置代理（推荐开启）」区域
  3. 勾选/取消勾选「使用内置 CF Worker 代理」
  4. 设置自动保存

  --- 自定义 CF Worker（可选）---
  如果你想自己部署 CF Worker（最推荐 APK 用户自己部署，稳定性最好），
  可以填入自己的 Worker 地址，自定义代理会作为内置代理的备选。

  部署自己的 Cloudflare Worker 步骤（免费，无需服务器）：

  === 第一步：注册 Cloudflare ===
  1. 打开 https://dash.cloudflare.com/sign-up
  2. 用邮箱注册账号，不需要绑信用卡
  3. 建议手机端用谷歌商店注册谷歌账号再注册 CF，被封了申诉基本会成功（机器人审核，申诉即过）
  4. 注册完成后自动进入 Cloudflare 仪表盘

  === 第二步：创建 Worker ===
  1. 左侧菜单「Workers 和 Pages」→ 点「创建应用程序」→「创建 Worker」
  2. 给 Worker 起个名字，例如：xhs-proxy
  3. 点击「部署」

  === 第三步：编辑代码 ===
  1. 点击「编辑代码」，把默认代码全删掉，粘贴以下代码：

------------------------------------------------------------
export default {
  async fetch(request) {
    if (request.method === 'OPTIONS') {
      return new Response(null, { status: 204, headers: {
        'Access-Control-Allow-Origin': '*',
        'Access-Control-Allow-Methods': 'GET,OPTIONS',
        'Access-Control-Allow-Headers': '*',
        'Access-Control-Max-Age': '86400'
      }});
    }
    const url = new URL(request.url);
    const target = url.searchParams.get('url');
    if (!target) return new Response('xhs-proxy ready. Usage: ?url=<xhs_url>', {
      status: 200,
      headers: { 'Access-Control-Allow-Origin': '*', 'Content-Type': 'text/plain; charset=utf-8' }
    });
    try {
      const resp = await fetch(target, {
        headers: {
          'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/120.0.0.0 Safari/537.36',
          'Referer': 'https://www.xiaohongshu.com/',
          'Accept': '*/*'
        }
      });
      const body = await resp.arrayBuffer();
      const headers = new Headers(resp.headers);
      headers.set('Access-Control-Allow-Origin', '*');
      headers.set('Access-Control-Allow-Methods', 'GET, OPTIONS');
      headers.set('Access-Control-Allow-Headers', '*');
      headers.delete('content-security-policy');
      headers.delete('x-frame-options');
      return new Response(body, { status: resp.status, headers });
    } catch (e) {
      return new Response('fetch error: ' + e.message, {
        status: 502,
        headers: { 'Access-Control-Allow-Origin': '*', 'Content-Type': 'text/plain' }
      });
    }
  }
}
------------------------------------------------------------

  2. 点击「部署」
  3. 复制 Worker 网址（格式：https://xhs-proxy.你的用户名.workers.dev）
  4. 回到插件「设置」标签页，粘贴到「自定义 Cloudflare Worker 代理地址」
  5. 点击「保存设置」


📌 六、手动注入

  不想用自动监听，也可以手动粘贴链接注入：
  1. 在主面板「手动注入」输入框粘贴小红书链接
  2. 在下拉框选择要注入到的目标会话
  3. 点击「注入」
  4. 注入完成后目标会话中会出现笔记内容


📌 七、预览功能

  在注入之前想先看内容，可以点「预览抓取内容」按钮，
  会在输入框下方显示抓取结果（不注入到聊天中）。


📌 八、日志查看

  遇到问题时，切换到「日志」标签页：
  - 可以看到每次抓取、注入、下载图片的详细记录
  - 点击「复制全部日志」可以复制出来发给别人帮忙排查


📌 九、注意事项

  1. 插件关闭面板后监听继续运行，如需停止，取消勾选「启用自动监听」
  2. 最多同时监听 3 个会话
  3. 注入后原消息会被替换，你发的链接+文字会消失，变成笔记内容
  4. 模式一只注入最多 9 张图片
  5. 如果内置代理不可用，可以在设置中关闭，使用公共代理或自己的 Worker
  6. 不需要保持插件面板打开，关闭面板监听照常工作
  7. Cloudflare Worker 完全免费，每天有 10 万次请求额度（内置代理也是免费的）


📌 十、常见问题

  Q: 注入后聊天界面没刷新？
  A: 切到另一个聊天再切回来即可看到。插件会尝试自动刷新（Pinia splice
     → 事件派发 → viewStack 强制刷新三方案降级），极端情况手动切一下。

  Q: 图片下载失败？
  A: 浏览器端请确保内置代理开关开启（默认开启）。如果内置代理不可用
     （极少数网络环境），可以自己部署 CF Worker 填入自定义地址。

  Q: 浏览器打开 HTML 时 corsproxy 返回 403？
  A: 这是 corsproxy.io 免费版的政策：仅允许 localhost/GitHub.io 等
     Origin 白名单。v2.6.1 已内置官方代理解决此问题，请确保内置代理
     开关为开启状态（设置中第一个开关，默认开启）。

  Q: APK 中能用 corsproxy，浏览器中不能？
  A: APK WebView 不发 Origin header，corsproxy 认为"服务器端请求"放行。
     浏览器强制发 Origin，corsproxy 检查白名单返回 403。
     v2.6.1 内置代理已解决此问题。

  Q: 内置代理安全吗？我的隐私数据会被看到吗？
  A: 内置代理仅处理小红书公开数据（笔记链接、HTML页面、图片）。
     你的聊天内容、角色人设、API Key 等隐私数据完全在你本地设备处理，
     不经过任何代理服务器。如果仍不放心，可以关闭内置代理，
     自己部署 CF Worker 填入自定义地址。

  Q: 副 API 连接测试失败？
  A: 检查 Base URL 是否正确（注意不要多/少 /v1），
     API Key 是否还有余额，模型名是否拼写正确。

  Q: workers.dev 在国内访问不了？
  A: 部分网络环境下 workers.dev 可能被阻断。如果遇到此问题：
     - APK 端不受影响（走 corsproxy 公共代理）
     - 浏览器端可以自己部署 CF Worker 并绑定自定义域名
       （需要一个域名，将 NS 指向 Cloudflare 即可，国内可正常访问）
     - 或者最简单：用 APK 端使用本插件


📌 十一、环境差异说明

  本插件针对两种运行环境做了优化：

  1. APK 端（Android WebView）：
     - 不发 Origin header，corsproxy 公共代理可用
     - 不需要 CF Worker 也能正常抓取
     - 不需要做任何配置，开箱即用

  2. 浏览器端（Chrome/Edge/Safari 等）：
     - 强制发 Origin header，corsproxy 必然返回 403
     - 必须使用 CF Worker（内置或自定义）
     - v2.6.1 起默认开启内置代理，开箱即用

  内置代理开关只影响浏览器端，APK 端不受此开关影响。
